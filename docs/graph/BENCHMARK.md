# 그래프 탐색 벤치마크 결과

## 테스트 환경

- **CPU**: 제한 환경 (1~4 cores)
- **Memory**: 256MB ~ 1GB
- **Java**: 21
- **Spring Boot**: 3.5.0

## 2. 성능 분석

### 시간복잡도 검증

모든 알고리즘이 **O(V + E)**이므로 비슷한 성능을 보임
```
1,000 노드:   ~0.8ms
100,000 노드: ~40ms

증가율: 100배 노드 → 50배 시간 증가 (선형에 가까움)
```

### 알고리즘별 특성

| 알고리즘 | 소규모 (1K) | 대규모 (100K) | 특징 |
|---------|------------|--------------|------|
| DFS (Recursive) | 🥇 최고 | 4위 | 깊은 그래프에서 스택 오버플로우 위험 |
| DFS (Iterative) | 4위 | 🥈 2위 | 명시적 스택으로 안전 |
| BFS | 3위 | 🥉 3위 | 최단 경로 보장 |
| Bidirectional BFS | 2위 | 🥇 최고 | 양방향 탐색으로 효율적 |

### 메모리 사용량
```
100,000 노드 기준:
- DFS (Recursive): 17.8MB - 재귀 스택
- DFS (Iterative): 18.0MB - 명시적 스택
- BFS:             22.0MB - 큐가 넓어질 수 있음
- Bidirectional:   19.0MB - 양방향 자료구조
```

## 4. 결론 및 선택 가이드

### 상황별 추천

| 상황 | 추천 알고리즘 | 이유 |
|-----|-------------|------|
| **최단 경로 필요** | **BFS** | 가중치 없는 그래프에서 최단 보장 |
| 모든 경로 탐색 | DFS | 백트래킹에 적합 |
| 깊은 그래프 | DFS (Iterative) | 스택 오버플로우 방지 |
| 두 노드 간 경로 | Bidirectional BFS | 탐색 범위 감소 |
| 사이클 탐지 | DFS | 방문 상태 추적 용이 |
| 위상 정렬 | DFS | 후위 순회 활용 |

### 핵심 정리
```
최단 경로가 필요하면 → BFS
그 외에는 → DFS (대부분의 경우 비슷한 성능)
대규모 그래프 → DFS (Iterative) 권장
```
