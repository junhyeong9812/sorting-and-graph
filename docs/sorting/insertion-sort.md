# 삽입 정렬 (Insertion Sort)

## 요약

| 항목 | 내용 |
|-----|------|
| 시간복잡도 (최선) | O(n) - 이미 정렬된 경우 |
| 시간복잡도 (평균) | O(n²) |
| 시간복잡도 (최악) | O(n²) - 역순 정렬된 경우 |
| 공간복잡도 | O(1) |
| 안정성 | 안정 정렬 |

## 원리

카드를 정렬하는 방식과 같습니다.
손에 든 카드(정렬된 부분)에 새 카드를 올바른 위치에 삽입합니다.

## 동작 과정
```
초기: [5, 3, 8, 4, 2]
      [5] | 3, 8, 4, 2   ← 5는 이미 정렬됨

1회전: 3을 [5]에 삽입
  3 < 5 이므로 5 앞에 삽입
  [3, 5] | 8, 4, 2

2회전: 8을 [3, 5]에 삽입
  8 > 5 이므로 맨 뒤에 유지
  [3, 5, 8] | 4, 2

3회전: 4를 [3, 5, 8]에 삽입
  4 < 8 → 8 이동
  4 < 5 → 5 이동
  4 > 3 → 3 뒤에 삽입
  [3, 4, 5, 8] | 2

4회전: 2를 [3, 4, 5, 8]에 삽입
  2 < 8, 5, 4, 3 → 모두 이동
  맨 앞에 삽입
  [2, 3, 4, 5, 8]

결과: [2, 3, 4, 5, 8]
```

## 코드
```java
public void sort(int[] arr) {
    int n = arr.length;

    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;

        // key보다 큰 원소들을 오른쪽으로 이동
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }

        // key를 올바른 위치에 삽입
        arr[j + 1] = key;
    }
}
```

## 특징

**장점**
- 거의 정렬된 데이터에서 매우 빠름 (O(n))
- 안정 정렬
- 온라인 알고리즘 (데이터가 들어오는 대로 정렬 가능)
- 소규모 데이터에서 퀵 정렬보다 빠를 수 있음

**단점**
- 역순 데이터에서 최악 O(n²)
- 대규모 데이터에 부적합

**사용 시기**
- 데이터가 거의 정렬되어 있을 때
- 데이터가 실시간으로 들어올 때
- 소규모 데이터 (n < 50)
- 하이브리드 정렬의 부분 알고리즘 (TimSort 등)
